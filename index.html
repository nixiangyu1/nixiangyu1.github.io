序列化的作用

将内存中的实例化对象保存在磁盘中

序列化的使用场景

  1. 想把内存中的对象状态保存到一个文件或者数据库中
  2. 想用套接字在网络上传送对象
  3. 想通过RMI远程方法调度传输对象的时候

序列化需要注意的地方

  1. static修饰的静态变量和transient修饰的临时变量不会被序列化
  2. 父类实现序列化，子类自动实现序列化，子类是实现序列化，父类需要一个默认的无参构造函数

把本地仓库推送到远端

1.通过密匙建立连接

  1. 首先创建一个本地仓库
  2. 打开 git Bash 输入ssh-keygen -t rsa，一直回车
  3. c盘的当前用户下的.ssh文件夹下，创建公钥和私钥，没有后缀的是私钥
  4. 复制有后缀的公钥,打开github的settings点击SSH and GPG keys添加公钥
  5. 重新打开git Bash，输入 git remote add origin git@github.com:nixiangyu1/repo1.git
  6. 将本地仓库推送到远程 git push -u origin master
